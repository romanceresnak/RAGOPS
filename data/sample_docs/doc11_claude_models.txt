Title: Claude Model Family on Amazon Bedrock — Capabilities and Selection Guide

Anthropic's Claude model family is available on Amazon Bedrock in multiple tiers, each optimised for different latency, cost, and capability requirements.

Claude 3.5 Sonnet (claude-3-5-sonnet-20241022-v2:0):
- Context window: 200,000 tokens
- Output limit: 8,192 tokens
- Strengths: Complex reasoning, code generation, nuanced analysis, RAG synthesis
- Prompt caching: Yes (minimum 1,024-token cache block)
- Input price: $3.00/1M | Output: $15.00/1M
- Best for: Production RAG, CAG, complex KAG reasoning

Claude 3 Haiku (claude-3-haiku-20240307-v1:0):
- Context window: 200,000 tokens
- Output limit: 4,096 tokens
- Strengths: Fast, cheap, good for simple Q&A and classification
- Prompt caching: Yes
- Input price: $0.25/1M | Output: $1.25/1M
- Best for: High-volume simple queries, routing classifier, batch processing

Claude 3 Opus (claude-3-opus-20240229-v1:0):
- Context window: 200,000 tokens
- Strengths: Highest capability for complex multi-step tasks
- Input price: $15.00/1M | Output: $75.00/1M
- Best for: Complex agentic tasks where accuracy > cost

Key features relevant to RAG/CAG/KAG:
- All Claude models support system prompts (ideal for CAG's pre-loaded knowledge base)
- Streaming (InvokeModelWithResponseStream) reduces perceived latency
- Claude models return usage.input_tokens, usage.output_tokens, usage.cache_read_input_tokens, and usage.cache_creation_input_tokens in every response — enabling precise cost tracking
- Tool use (function calling) enables agentic RAG where Claude decides which knowledge source to query

Model selection strategy:
Query complexity score 1–3 → Haiku
Query complexity score 4–7 → Sonnet
Query complexity score 8–10 → Opus
Batch jobs → Haiku or Batch Inference (50% discount)
